## API Reference

### StarRating

| Prop        | Type                      | Default     | Description                                                                                                      |
| ----------- | ------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------- |
| `value`     | `number`                  | Required    | The current rating value. Can be decimal to display partial stars.                                               |
| `onChange`  | `(value: number) => void` | -           | Callback called when the user changes the rating. If not provided, the component will operate in read-only mode. |
| `className` | `string`                  | -           | Custom CSS class to apply to the container.                                                                      |
| `size`      | `number`                  | `24`        | Size of the star icons in pixels.                                                                                |
| `maxStars`  | `number`                  | `5`         | Maximum number of stars to display.                                                                              |
| `readOnly`  | `boolean`                 | `false`     | If `true`, the component will be in read-only mode and non-interactive.                                          |
| `color`     | `string`                  | `"#e4c616"` | Color of the stars (CSS color format).                                                                           |

## Accessibility

The StarRating component can be interacted with using a mouse and supports:

- Visual feedback when hovering over stars
- Accurate rating display using gradients for decimal values
- Color transitions and hover effects to improve UX

## Best Practices

1. **Use readOnly mode** when displaying ratings from other users or system data.

2. **Customize colors** to match your application's theme.

3. **Validate input values** when used in forms. The `value` should be within the range of 0 to `maxStars`.

4. **Provide visual feedback** to let users know they have successfully changed a rating.

5. **Consider appropriate sizing** for your design and responsive UI.
